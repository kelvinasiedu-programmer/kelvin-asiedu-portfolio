PulseCommerce

Python, DuckDB, scikit-learn, XGBoost, Streamlit, Docker | Mar 2026 to Apr 2026

Most ecommerce teams have clickstream data they cannot act on. PulseCommerce turns 450K raw events into four answers: where the funnel leaks, which customers will churn, what demand looks like next month, and whether the last A/B test actually won.

  • Transformed 450K raw ecommerce events into 5 decision-ready analytics layers covering funnel diagnosis, demand forecasting, churn scoring, and A/B test evaluation.
  • Trained an XGBoost churn-risk model that reached 0.889 AUC on customer behavior features.
  • Authored 13 pytest cases covering 68% of the codebase and containerized the project with Docker.
  • Built a stakeholder-facing Streamlit dashboard that translated technical outputs into usable business KPIs.
  • Churn model surfaces the top-decile at-risk cohort, so a retention team using it focuses outreach on roughly 10% of the base instead of spraying discounts across the file.

RAG Chatbot

Python, FastAPI, FAISS, Next.js, TypeScript, Docker | Oct 2025 to Nov 2025

Enterprise LLM rollouts stall on one question: can I trust the answer? This chatbot ships every response with the source document it came from, removing the trust friction that kills internal AI adoption.

  • Engineered an end-to-end RAG pipeline that combined FAISS vector retrieval with LLM generation.
  • Reduced hallucination rate through prompt engineering, embedding tuning, and response evaluation loops.
  • Deployed a full-stack application with a FastAPI backend and a Next.js frontend aligned with enterprise AI delivery patterns.
  • Grounded-citation design means a reviewer sees exactly which uploaded PDF an answer came from, so the tool is auditable instead of a black box.

Bank Account System

Python, FastAPI, Pydantic, Docker, pytest | Mar 2025 to May 2025

A production-pattern banking backend: OOP domain model, tiered-interest savings, overdraft-guarded checking, 38 tests, FastAPI + Docker. Built to prove I can ship the kind of service a real bank would deploy, not a tutorial clone.

  • Architected an OOP banking platform with Savings and Checking inheritance, tiered interest logic at 3%, 5%, and 7%, plus overdraft validation.
  • Achieved 38 passing unit tests covering balance validation, transaction auditing, and account interactions.
  • Exposed the system through a FastAPI REST API with Pydantic schema validation and Docker deployment.
  • Overdraft and tiered-interest rules are enforced at the schema layer, so a downstream API consumer cannot create an invalid account state.
